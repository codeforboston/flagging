Source Downloads

Due to free tier hosting limitations, we cannot store lots of data at a time in the database.

However, the data processing functions can be run directly and sent as a download without ever touching the database. This allows for sending larger chunks of data at a time because it is not bound by limitations on the number of rows we can store imposed by our hosting service.

The downsides to this approach are (1.) it takes a while, and (2.) it may consume a lot of bandwidth and CPU time, so use it sparingly!

Note: Some of the earliest data in the "Processed Data" and "Model Outputs" tables will be missing or distorted due to some data transformations that require looking backwards in time, e.g. rolling averages or time since last significant rainfall. It is suggested that you discard the earliest month of data for each download.
